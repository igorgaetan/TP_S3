Étapes de déploiement
01
AWS Créer un bucket S3

Créez un bucket S3 avec un nom identique à votre domaine (ex : mon-site.com). Décochez "Bloquer tout accès public" pour permettre l'accès web. 

aws s3api create-bucket \
  --bucket mon-bootcamp-devops \
  --region eu-west-1 \
  --create-bucket-configuration LocationConstraint=eu-west-1
02
S3 Activer le Static Website Hosting

Dans les propriétés du bucket, activez "Static website hosting", définissez index.html comme document index et error.html comme page d'erreur. 

aws s3 website s3://mon-bootcamp-devops \
  --index-document index.html \
  --error-document error.html
03
IAM Configurer la Bucket Policy

Appliquez une politique IAM pour autoriser la lecture publique de tous les objets du bucket. 

{
  "Version": "2012-10-17",
  "Statement": [{
    "Effect":    "Allow",
    "Principal": "*",
    "Action":    "s3:GetObject",
    "Resource":  "arn:aws:s3:::mon-bootcamp-devops/*"
  }]
}
04
Upload Déployer les fichiers

Uploadez votre index.html (et assets) vers le bucket via la CLI. Le flag --acl peut être omis si la bucket policy est correcte. 

# Upload d'un fichier unique
aws s3 cp index.html s3://mon-bootcamp-devops/

# Sync d'un dossier entier (recommandé)
aws s3 sync ./dist s3://mon-bootcamp-devops \
  --delete
05
HTTPS Configurer CloudFront (CDN + TLS)

Créez une distribution CloudFront pointant vers l'endpoint S3. Activez la redirection HTTP→HTTPS. Ajoutez un certificat ACM pour votre domaine. 

resource "aws_cloudfront_distribution" "site" {
  origin {
    domain_name = aws_s3_bucket.site.bucket_regional_domain_name
    origin_id   = "S3-mon-bootcamp"
  }
  default_cache_behavior {
    viewer_protocol_policy = "redirect-to-https"
  }
}
06
DNS Pointer le domaine (Route 53)

Créez un enregistrement A (Alias) ou CNAME dans Route 53 pointant vers votre distribution CloudFront. 

# Enregistrement A Alias (recommandé)
Name  : mon-site.com
Type  : A — Alias
Value : d1234abcd.cloudfront.net

# Sous-domaine www → redirect vers apex
Name  : www.mon-site.com
Type  : CNAME
Value : mon-site.com
07
CI/CD Automatiser avec GitHub Actions

Configurez un pipeline qui déploie automatiquement à chaque push sur main, invalide le cache CloudFront, et notifie l'équipe. 

on:
  push:
    branches: [main]

jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - name: Deploy to S3
        run: |
          aws s3 sync ./dist s3://${{ secrets.S3_BUCKET }} --delete
          aws cloudfront create-invalidation \
            --distribution-id ${{ secrets.CF_DIST_ID }} \
            --paths "/*"
        env:
          AWS_ACCESS_KEY_ID:     ${{ secrets.AWS_ACCESS_KEY_ID }}
          AWS_SECRET_ACCESS_KEY: ${{ secrets.AWS_SECRET_ACCESS_KEY }}
